## Cursor Rules (Enforced)

All development in this repository MUST follow these rules.

### Scope + discipline

- Do not expand scope.
- Do not introduce new architecture patterns.
- Keep implementations minimal.
- Break large features into smaller working steps.
- Avoid overengineering.

### Before completing any major feature

- Confirm build passes.
- Confirm TypeScript strict mode passes (if TypeScript is present in the repo).
- Confirm no unused code.
- Confirm no feature drift.
- Do not claim something is “fixed” unless you have verified it (run the app / reproduce the issue / confirm the expected behavior).
